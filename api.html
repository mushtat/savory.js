Callbacks and API

Currently there are two callbacks in Savory.js:

  • onReady - executed when Savory.js is initiated
  • onLoad - executed when the new content is loaded, parsed and displayed API

API

When Savory is inited (automatically or manually) - it can be reached at window._savory .

In window._savory object are some informational properties::

  • base - contains info of the site origin
  • config - stores configuration object
  • interface - contains document.location info and inner modules (parser, error and loader)

Moreover, there is actually no API in the wide sense, just some info properties ¯\_(ツ)_/¯
