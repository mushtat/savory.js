Scripts loading and execution

For correct scripts handling in Savory.js all scripts must be appended into the container with id "scripts" (on default). In the other case they will be not executed while loading with page content.

Every script in the scripts container is moved to a head element (in the other case script will not be executed) and automatically executed. On page change these scripts are removed. For placing and identification of scripts is used the ideology similar to require.js and own realizing.

Here is simple code of page for load with script:

<!--<title>Page title</title>-->
<div id="container">
    Page content
</div>
<div id="scripts">
    <script type="text/javascript">
        console.log('Sample script');
    </script>
</div>

You can manually set id of script container node in savory config:

<script type="text/javascript">
    window.savoryConfig = {
        container : 'container', // id of main container
        scriptsContainer : 'scripts', // id of scripts container
    }
</script>
