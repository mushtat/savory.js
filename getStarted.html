Overview

The framework includes the transparent ajax transitions between pages for user. Web-site or application with Savory enabled actually is one html page with reloadable content and titles. First of all Savory framework directs the users with the low and mediate html/css coding skills but who needs the whole one page application advantage (such as background tasks or music player) without page reloading. For complex single-page applications, the author recommends to use highly specialized tools, like ember.js, angular.js, backbone, react or similar libs and frameworks. Savory.js is suitable for mobile application development with Apache Cordoba, PhoneGap, etc.

Important: Savory.js's aim is to remove gap between a simple task and a complex solution. It is the simplest way to make your static site AJAX-based

Savory itself does not depend on the other libs (like jQuery or Zepto). The minified file contains the library nanoajax to see it you need to use a microscope and html5 histiory api expansion.

For build there is Browserify used with few development dependencies. More info you can get on the building info page.

Browser compatibility:

  • Google Chrome
  • Mozilla Firefox
  • Safari
  • Opera
  • Internet Explorer 9+
  • IOS и Android browser
