Work with links in Savory.js

Framework should know when the user navigates the site or leaves it. Theoretically, it can be automated by comparing the current site href and the target link href origin But it is rather imperfect. Also there can be situations when external link must be setted manually. Here is the information about this question.

There are two types of links - external and internal. On default all links on the page are perceived as the internal ones and are loaded by Savory without page reloading. The external links are recognized by the class "external". This class name can be configured in Savory initializing config. You can read about this on API page. Default behaviour can be switched over to completely opposite - all the links became external and classname defined in Savory config used to determine internal links

Savory config examples:

Switch default link recognizer (from "all links are internal" to "all links are external")

<script type="text/javascript">
    window.savoryConfig = {
        links : {
            external : true
        }
    }
</script>

Set classname for links to ignore

<script type="text/javascript">
    window.savoryConfig = {
        links : {
            identificatorClassName : 'external' //classname to identificate ignored link nodes
        }
    }
</script>

Default links config

<script type="text/javascript">
    window.savoryConfig = {
        links : {
            external : false
            identificatorClassName : 'external' //classname to identificate ignored link nodes
        }
    }
</script>
