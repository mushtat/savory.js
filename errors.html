Errors handling

In case when a loaded page status code is not 200 an error is showed. A simple error message will appear in the left top corner.

When you click on this link - you will see Savory.js error.

In case you want to set your own error page - you have to use server-side url recognition and return an error page content with 200 status. The author understands that this behaviour probably is not appropriate and is waiting a feedback from the server-side programmers commenting on github.
